# Базовый пример миграции

Этот пример демонстрирует простейший сценарий миграции репозитория с SourceCraft на GitHub.

## 📋 Сценарий

- **Репозиторий**: Простой проект с одной веткой (main)
- **Размер**: Малый (< 100MB)
- **История**: Линейная, без сложных мерджей
- **Особенности**: Нет подмодулей, нет LFS

## 🚀 Быстрый старт

### 1. Подготовка переменных

Создайте файл `.env` в этой папке:

```bash
# examples/basic-migration/.env
SOURCE_CRAFT_ORG="ваша_организация"
SOURCE_CRAFT_REPO="простой-репозиторий"
GITHUB_USERNAME="ваш_github_username"
GITHUB_REPO="простой-репозиторий"
```

### 2. Запуск миграции

```bash
# Перейдите в папку с примером
cd examples/basic-migration

# Сделайте скрипт исполняемым
chmod +x migrate.sh

# Запустите миграцию
./migrate.sh
```

## 📁 Структура примера

```
basic-migration/
```

## 🛠️ Скрипты

### migrate.sh
Основной скрипт миграции, который:
1. Проверяет предварительные условия
2. Зеркально клонирует репозиторий
3. Отправляет на GitHub
4. Создает лог выполнения

### verify.sh
Скрипт проверки, который:
1. Сравнивает хеши коммитов
2. Проверяет количество веток и тегов
3. Создает отчет о миграции

### cleanup.sh
Скрипт очистки, который:
1. Удаляет временные файлы
2. Очищает логи (по желанию)
3. Восстанавливает исходное состояние

## 📊 Ожидаемый результат

После успешной миграции вы получите:

1. **Точную копию** репозитория на GitHub
2. **Лог выполнения** в папке `logs/`
3. **Отчет о проверке** в консоли
4. **Чистую рабочую область** (без временных файлов)

## 🧪 Тестовый прогон

Для тестового прогона без реальной миграции:

```bash
# Режим dry-run (проверка без реальных действий)
./migrate.sh --dry-run

# Только проверка
./verify.sh --source-only
```

## ⚠️ Предупреждения

1. **Токены доступа**: Убедитесь, что у вас есть действительные токены
2. **Права доступа**: Проверьте права на чтение/запись
3. **Интернет соединение**: Убедитесь в стабильности соединения
4. **Дисковое пространство**: Убедитесь в наличии свободного места

## 📝 Логирование

Все действия логируются в:
- `logs/migration-YYYY-MM-DD-HH-MM-SS.log` - полный лог
- `logs/error.log` - только ошибки
- Консольный вывод с цветовой маркировкой

## 🆘 Поиск проблем

Если возникли проблемы:

1. Проверьте логи: `tail -f logs/error.log`
2. Проверьте токены: `echo $SOURCE_CRAFT_TOKEN`
3. Проверьте URL: `./migrate.sh --test-urls`
4. Обратитесь к документации в `docs/troubleshooting.md`
